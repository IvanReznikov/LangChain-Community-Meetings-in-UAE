üîß What are Tools?

Tools are functions or capabilities that LLMs can use to interact with external systems.

Key Concepts:

  • Extend LLM capabilities beyond text generation
  • Enable interaction with APIs, databases, and external services
  • Structured input/output interfaces
  • Can be custom-built or pre-made

Example Tool Definition:

from langchain.tools import Tool from langchain.tools import DuckDuckGoSearchRun # Creating a search tool search = DuckDuckGoSearchRun() search_tool = Tool( name="Search", func=search.run, description="Useful for searching the internet for current information" ) # Custom tool example def calculate(expression: str) -> str: """Evaluates a mathematical expression""" try: return str(eval(expression)) except: return "Error in calculation" calc_tool = Tool( name="Calculator", func=calculate, description="Useful for mathematical calculations" )

üìû Tool Calling Agent

Modern approach where the LLM directly generates tool calls in a structured format.

How It Works:

  • LLM has native understanding of function/tool calling
  • Generates structured JSON for tool invocation
  • Parallel tool execution support
  • Built-in parameter validation
from langchain.agents import create_tool_calling_agent from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate # Create prompt prompt = ChatPromptTemplate.from_messages([ ("system", "You are a helpful assistant. Use tools when needed."), ("placeholder", "{chat_history}"), ("human", "{input}"), ("placeholder", "{agent_scratchpad}"), ]) # Create tool-calling agent llm = ChatOpenAI(model="gpt-4", temperature=0) agent = create_tool_calling_agent(llm, tools, prompt) # Execute with AgentExecutor from langchain.agents import AgentExecutor agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True ) # Run the agent result = agent_executor.invoke({ "input": "Search for LangChain tutorials and calculate 42 * 17" })

üí° Strengths

  • Explicit reasoning steps
  • Better interpretability
  • Works with any LLM
  • Flexible prompting

üîç Best For

  • Complex reasoning tasks
  • Debugging workflows
  • Educational purposes
  • Older LLMs

Aspect Tool Calling Agent ReAct Agent
Approach Direct function calling via LLM Thought ‚Üí Action ‚Üí Observation loop
LLM Requirements Modern LLMs with tool support Any LLM with text generation
Reasoning Implicit (internal to LLM) Explicit (visible in output)
Performance Faster, fewer tokens More tokens, slower
Debugging Less transparent Highly transparent
Parallel Execution ‚úÖ Native support ‚ùå Sequential only
Error Recovery Built-in retry mechanisms Manual handling needed

When to Use Which?

Choose Tool Calling when:

  • Using modern LLMs (GPT-4, Claude 3+)
  • Need efficient, production-ready agents
  • Want parallel tool execution
  • Prioritize speed and token efficiency

Choose ReAct when:

  • Need explicit reasoning traces
  • Working with older or open-source LLMs
  • Debugging complex workflows
  • Teaching or demonstrating agent behavior

# MCP Server - Provides tools to clients from mcp.server import Server from mcp.types import Tool, TextContent app = Server("my-tools-server") @app.tool() async def get_weather(location: str) -> TextContent: """Get weather for a location""" # Implementation here return TextContent(text=f"Weather in {location}: Sunny, 72¬∞F") @app.tool() async def search_database(query: str) -> TextContent: """Search internal database""" # Implementation here return TextContent(text=f"Results for: {query}") # Run server if __name__ == "__main__": import asyncio from mcp.server.stdio import stdio_server asyncio.run(stdio_server(app))

MCP Client Integration:

# MCP Client - Connects to MCP servers from langchain_mcp import MCPToolkit import asyncio async def main(): # Connect to MCP server toolkit = MCPToolkit( server_command=["python", "my_mcp_server.py"] ) # Get tools from server tools = toolkit.get_tools() # Use with LangChain agent from langchain.agents import create_tool_calling_agent agent = create_tool_calling_agent( llm=llm, tools=tools, prompt=prompt ) # Execute result = await agent.ainvoke({ "input": "What's the weather in Paris?" }) asyncio.run(main())

üöÄ Transport Options

  • stdio - Standard I/O

üîß Server Features

üåü Key Advantages:

MCP servers are language-agnostic and can be written in Python, JavaScript, or any language that supports the protocol. Once deployed, they can be used by any MCP-compatible client.

  • Write once, use everywhere
  • No vendor lock-in
  • Community-driven tool ecosystem
  • Seamless integration with LangChain
